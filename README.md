# AI-редактор Telegram-канала (content-bot)

Telegram-бот, который ведёт закрытый канал по календарю контента: присылает владельцу
задание на пост, задаёт уточняющие вопросы, принимает текст/фото/видео, генерирует
черновик через Claude API, отправляет на согласование и публикует в канал
**только после явного одобрения владельца**. Есть простая web-админка.

## Главное правило

Бот **никогда** не публикует автоматически. Публикация происходит только после
явного одобрения: нажатие кнопки «✅ Одобряю» или текст «одобряю / публикуй /
можно публиковать / утверждаю». Расписание не имеет приоритета над согласованием —
если время пришло, а пост не одобрен, бот лишь напоминает владельцу.

## Стек

Python 3.12, aiogram 3, FastAPI, SQLAlchemy 2 (async) + asyncpg, Alembic, APScheduler,
Jinja2, Anthropic SDK, Docker Compose. Абстрактный AI-клиент позволяет заменить
провайдера (OpenAI, DeepSeek и т.д.).

## 1. Установка

```bash
git clone <repo> content-bot && cd content-bot
cp .env.example .env
```

## 2. Заполнение `.env`

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | токен бота от @BotFather |
| `OWNER_TELEGRAM_ID` | ваш Telegram ID (узнать: @userinfobot) |
| `ADMIN_USERNAME` / `ADMIN_PASSWORD` | вход в админку |
| `DATABASE_URL` | для Docker Compose оставьте как есть |
| `ANTHROPIC_API_KEY` | ключ Claude API (console.anthropic.com) |
| `AI_MODEL` | по умолчанию `claude-sonnet-5` |
| `DEFAULT_CHANNEL_ID` | ID закрытого канала (см. п.6) |
| `BOT_MODE` | `polling` локально, `webhook` на проде |
| `WEBHOOK_URL` | публичный URL приложения (только для webhook) |
| `WEBHOOK_SECRET` | произвольная секретная строка |
| `TIMEZONE`, `DAILY_CHECK_TIME` | таймзона и время ежедневной проверки |

Секреты хранятся только в `.env`, в базу и в интерфейс не попадают.

## 3. Запуск через Docker Compose

```bash
docker compose up --build
```

Миграции применяются автоматически при старте. Приложение — на `http://localhost:8000`,
health-check — `http://localhost:8000/health`.

Для локальной разработки используйте `BOT_MODE=polling` — публичный адрес не нужен.

## 4. Создание бота через BotFather

1. Откройте @BotFather → `/newbot`, задайте имя и username.
2. Скопируйте токен в `BOT_TOKEN`.

## 5. Добавление бота администратором канала

1. Создайте закрытый канал.
2. Настройки канала → Администраторы → добавьте бота.
3. Дайте право «Публикация сообщений».

## 6. Как узнать channel_id

- Перешлите любое сообщение из канала боту @userinfobot, либо
- добавьте @getidsbot в канал.
- ID закрытого канала выглядит как `-100XXXXXXXXXX`. Впишите его в
  `DEFAULT_CHANNEL_ID` или в админке (Настройки).

## 7. Админ-панель

Откройте `http://localhost:8000/admin`, войдите по `ADMIN_USERNAME`/`ADMIN_PASSWORD`.
Здесь: календарь, CRUD задач, включение/выключение, ручной запуск, просмотр версий
постов, медиа и логов согласования, настройки (владелец, канал, промт, модель).

## 8. Первая задача в календаре

В админке «Добавить задачу»: дата = сегодня, рубрика, тема, цель, описание → «Добавить».

## 9. Проверка генерации

Откройте задачу → «▶️ Запустить на сегодня» (или в боте `/today`). Бот пришлёт
задание и вопросы. Ответьте, при желании пришлите фото → «Медиа добавлено, генерируем».

## 10. Проверка согласования

Бот пришлёт черновик с кнопками. Проверьте:
- «✏️ Правки» и «🔄 Другой вариант» создают новые версии, но **не публикуют**;
- произвольный текст не считается одобрением;
- «❌ Отменить» переводит задачу в `cancelled`.

## 11. Проверка публикации

Нажмите «✅ Одобряю» (или напишите «одобряю»). Пост появится в канале, статус станет
`published`. Повторное «Одобряю» второй раз не публикует (защита от дублей).

## 12. Подключение Claude API (через AITunnel)

Проект использует прокси **AITunnel** (Anthropic-совместимый API, доступен из РФ).
В `.env`:

```
ANTHROPIC_API_KEY=sk-aitunnel-...        # ключ AITunnel
ANTHROPIC_BASE_URL=https://api.aitunnel.ru
AI_MODEL=claude-sonnet-5                  # проверено, работает через AITunnel
```

AITunnel принимает токен в заголовке `Authorization: Bearer`, поэтому при заданном
`ANTHROPIC_BASE_URL` код передаёт ключ через параметр `auth_token` SDK. Список
доступных моделей: `GET https://api.aitunnel.ru/v1/models`.

Для прямого доступа к Anthropic (без прокси) оставьте `ANTHROPIC_BASE_URL` пустым
и укажите ключ `sk-ant-...` — тогда используется стандартный `x-api-key`.

## 13. Замена AI-провайдера в будущем

Реализуйте класс-наследник `AIProvider` в `app/ai/providers.py` и зарегистрируйте
его фабрику в словаре `PROVIDERS`. Затем укажите имя провайдера в настройках/`.env`
(`AI_PROVIDER`). Остальной код менять не нужно.

## Деплой на Render

Файл `render.yaml` описывает web-сервис (Docker) + Postgres. При деплое задайте
секреты (`BOT_TOKEN`, `OWNER_TELEGRAM_ID`, ключи и т.д.), `BOT_MODE=webhook` и
`WEBHOOK_URL` = публичный адрес сервиса. Webhook ставится автоматически при старте.

## Что нужно дозаполнить (learning mode)

Две функции в `app/services/approval.py` намеренно оставлены для вас — это ядро
бизнес-логики согласования:
- `ALLOWED_TRANSITIONS` — словарь разрешённых переходов статусов;
- `is_text_approval()` — распознавание текстового одобрения.

Пока они не заполнены, бот не сможет менять статусы задач. См. подробные подсказки
в комментариях внутри файла.
